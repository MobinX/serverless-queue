# Getting Started

This guide walks you through installing the library and wiring up an email queue in **3 files** using the pre-built `DrizzlePgStorage` and `SimpleRetryStrategy`. The same pattern applies to any platform — see the [platform guides](./platforms/nextjs.mdx) for platform-specific details.

---

## Install

```bash
npm install serverless-queue drizzle-orm
# or
bun add serverless-queue drizzle-orm
# or
pnpm add serverless-queue drizzle-orm
```

> 📝 **Note:** `serverless-queue` has zero required runtime dependencies. `drizzle-orm` is only needed if you use the pre-built `DrizzlePgStorage` or `DrizzleSqliteStorage`.

---

## What you'll build

An email queue that sends transactional emails using Resend and Neon (serverless Postgres):

1. A message arrives and is persisted as `pending`
2. If the queue is idle it starts processing immediately
3. If the queue is busy the message waits and is drained automatically when the current job finishes
4. On failure the library retries up to `maxAttempts` times before calling `onExhausted`

## Quickstart — 3 files

### File 1: `lib/queue/action.ts` — your business logic

```ts
import { MessageAction } from 'serverless-queue'
import type { QueueMessage } from 'serverless-queue'
import { Resend } from 'resend'

interface EmailPayload { to: string; subject: string; html: string }

const resend = new Resend(process.env.RESEND_API_KEY!)

export class EmailAction extends MessageAction<EmailPayload> {
  async execute(messages: QueueMessage<EmailPayload>[]) {
    for (const msg of messages) {
      await resend.emails.send({ from: 'noreply@yourapp.com', ...msg.payload })
    }
  }
}
```

### File 2: `lib/queue/index.ts` — wire everything together

```ts
import {
  MessageQueue, MessageInvoker, SimpleRetryStrategy, DrizzlePgStorage, pgQueueMessages,
} from 'serverless-queue'
import type { QueueMessage } from 'serverless-queue'
import { neon } from '@neondatabase/serverless'
import { drizzle } from 'drizzle-orm/neon-http'
import { EmailAction } from './action'

const db = drizzle(neon(process.env.DATABASE_URL!))

class FetchInvoker extends MessageInvoker {
  async invoke(messages: QueueMessage[]) {
    const res = await fetch(`${process.env.NEXT_PUBLIC_APP_URL}/api/queue`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'x-queue-secret': process.env.QUEUE_SECRET!,
      },
      body: JSON.stringify(messages),
    })
    if (!res.ok) throw new Error(`Invoke failed: ${res.status}`)
  }
}

export const queue = new MessageQueue(
  new DrizzlePgStorage(db, pgQueueMessages),
  new EmailAction(),
  new SimpleRetryStrategy({
    shouldRetry: (_msg, err) => !(err instanceof Error && err.message.includes('validation')),
    onExhausted: async (msg) => console.error('[queue] exhausted', msg.id),
  }),
  new FetchInvoker(),
  { batchSize: 10, maxAttempts: 3, drainMode: 'individual' },
)

// Helper to enqueue a message
export async function enqueue(payload: { to: string; subject: string; html: string }) {
  await queue.handle([{
    id: crypto.randomUUID(),
    queueId: 'email',
    payload,
    attempt: 0,
    createdAt: Date.now(),
    type: 'new',
  }])
}
```

### File 3: `app/api/queue/route.ts` — the HTTP handler

```ts
import { queue } from '@/lib/queue'
import type { QueueMessage } from 'serverless-queue'
import { NextRequest, NextResponse } from 'next/server'

export async function POST(req: NextRequest) {
  if (req.headers.get('x-queue-secret') !== process.env.QUEUE_SECRET) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }
  const messages = await req.json() as QueueMessage[]
  await queue.handle(messages)
  return NextResponse.json({ ok: true })
}
```

## Database migration

Run this SQL once against your Postgres database:

```sql
CREATE TABLE IF NOT EXISTS queue_messages (
  id          TEXT    NOT NULL PRIMARY KEY,
  queue_id    TEXT    NOT NULL,
  message     TEXT    NOT NULL,
  timestamp   BIGINT  NOT NULL,
  state       TEXT    NOT NULL DEFAULT 'pending'
);
CREATE INDEX IF NOT EXISTS idx_queue_messages_queue_id_timestamp
  ON queue_messages (queue_id, timestamp);
```

Or using Drizzle Kit:

```bash
npx drizzle-kit push
```

See [Drizzle Storage](./drizzle-storage.mdx) for full Drizzle Kit integration instructions.

## Sending a message

From any server-side code:

```ts
import { enqueue } from '@/lib/queue'

await enqueue({
  to: 'user@example.com',
  subject: 'Welcome!',
  html: '<h1>Hello</h1>',
})
```
